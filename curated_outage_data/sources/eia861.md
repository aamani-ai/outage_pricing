# Source Note: EIA-861

Status: candidate Phase 2 source

## Role

EIA-861 is the first national utility/grid-context source for county-year
features.

Proposed roles:

- utility reliability context;
- utility-to-county customer/service-territory mapping;
- AMI and customer technology context;
- utility ownership and structure features;
- validation context against filed reliability metrics.

## Why It Matters

The current price engine estimates outage risk from EAGLE-I history. EIA-861 can
add utility-side context around that history.

Questions it may help answer:

- Do high-outage counties align with weak filed reliability metrics?
- Are outage patterns different by utility ownership class?
- Does AMI penetration or reporting maturity relate to observed outage data?
- Which utilities serve each county, and with what customer weights?

## Expected Grain

EIA-861 tables can vary by schedule.

Important grains:

- utility-year;
- utility-state-year;
- utility-county-year or service-territory rows where available;
- sector-year for some customer/technology fields.

## Proposed Join

Build utility-county crosswalk first:

```text
EIA utility/county customer records -> utility_county_crosswalk.parquet
```

Then aggregate utility features to county-year using customer weights where
defensible.

## Leakage Rules

Annual utility features must be lagged before use in forward-looking models.

Default rule:

```text
feature_year <= prediction_year - 1
```

If publication timing is unclear, use conservative lagging.

## Caveats

- EIA-861 reliability metrics are filed utility-level metrics, not county-level
  EAGLE-I event metrics.
- Major-event-day treatment may differ from our pricing interest.
- Service territory and customer allocation can be many-to-many.
- Missingness varies by utility type and field.

## First Decision Needed

Should EIA-861 be used first as descriptive context or model-eligible features?

Recommendation: descriptive context first. Promote fields to model-eligible only
after lag and missingness rules are validated.
