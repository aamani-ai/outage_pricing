# Learning Log

This folder captures feedback and reusable lessons from each curated-data phase.

Every phase should end with a learning note:

```text
phase_1_cause_attribution_feedback.md
phase_2_grid_condition_feedback.md
phase_3_forward_modeling_feedback.md
```

Current Phase 1 notes:

```text
phase1_fl_2017_2020_initial_pilot.md
phase1_pnnl_first_preview.md
event_catalog_gap_sensitivity.md
```

Learning notes should include:

- what worked;
- what failed;
- source caveats discovered;
- join rules that need revision;
- schema fields that were missing;
- validation checks that caught issues;
- what the next phase should inherit;
- what should be added to the broader data-analysis checks library.
