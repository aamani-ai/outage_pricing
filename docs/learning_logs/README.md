# Learning Logs

This folder is for general modeling lessons that are broader than the current
outage pricing implementation.

The `docs/dicsscssion/` folder captures project-specific discussions. This
folder captures reusable thinking frameworks that can later be moved into the
broader Learning directory if they prove useful.

## Current Logs

| File | Purpose |
|---|---|
| `hazard_event_modeling_goal_first.md` | Fundamental framework for event, magnitude, frequency, spatial-temporal modeling, and why the method should follow the decision goal. |
| `hazard_catalog_architecture_by_peril.md` | Common architecture for raw evidence, harmonized historical catalogs, stochastic event sets, forward-looking adjustments, and first detailed hail pass. |
| `loss_modeling_event_vs_continuous.md` | Fundamental distinction between discrete event losses, continuous/cumulative-effect losses, and hybrid state-dependent loss. |
| `loss_taxonomy_axes.md` | General taxonomy separating loss process, loss category, time/reporting, spatial aggregation, financial expression, and actionability. |
| `portfolio_event_catalogs_and_aggregation.md` | Portfolio-level framework for event catalogs, spatial dependence, ELT/YLT outputs, vendor cat models, and why portfolio aggregation should be a separate workstream. |
| `vendor_cat_models_and_climate_conditioning.md` | Research note on how vendor cat models appear to handle climate-conditioned views, what is public, what is unknown, and how our architecture should treat climate adjustment. |
